# Task1. Проверочный лист по безопасности данных

Этот файл служит текстовой основой для mindmap в `task1-data-security-mindmap.drawio`.

## Категории данных и риски

| Категория | Тип данных | Ключевой риск | Оценка | Обоснование |
| --- | --- | --- | --- | --- |
| Публичные данные | Каталоги ЖК и объектов на витрине продаж | Искажение данных | Значительный | Неверные характеристики, цены или статусы объектов вводят клиентов в заблуждение, бьют по продажам и репутации. |
| Публичные данные | Маркетинговые материалы, общие условия сервисов, публичные инструкции | Обесценивание данных | Незначительный | Устаревшая публичная информация снижает ценность данных, но обычно не приводит к прямому инциденту ИБ. |
| Публичные данные | Публичные контакты офисов, каналов поддержки, общие реквизиты | Потеря данных | Незначительный | Потеря неудобна для бизнеса и клиентов, но не ставит под угрозу ключевые активы компании. |
| Внутренние данные | Архитектурные схемы, карта интеграций, технические описания | Утечка данных | Значительный | Раскрывают атакующему карту систем, точки входа, интеграции и зависимые сервисы. |
| Внутренние данные | Операционные логи, метрики, служебные события без ПДн | Потеря данных | Значительный | Затрудняет расследование инцидентов, поиск ошибок и аудит изменений. |
| Внутренние данные | Внутренние отчёты, регламенты, рабочая аналитика | Некачественные данные | Значительный | Ошибочные внутренние данные приводят к неверным управленческим и техническим решениям. |
| Конфиденциальные данные | ПДн клиентов: Ф. И. О., телефон, email, данные бронирования | Утечка данных | Критический | Это персональные данные, их утечка создаёт прямой риск штрафов, жалоб и репутационного ущерба. |
| Конфиденциальные данные | ПДн собственников, история ЖКУ, обращения, данные личного кабинета | Утечка данных | Критический | В кейсе уже есть признак смешения данных в личном кабинете, поэтому риск подтверждён бизнес-проблемой. |
| Конфиденциальные данные | Анкеты и данные для онлайн-сделки, сведения об объектах и сделках | Искажение данных | Критический | Ошибки в таких данных влияют на юридически и финансово значимые процессы покупки недвижимости. |
| Секретные данные | Учётные данные администраторов, сервисных аккаунтов, секреты платформ | Утечка данных | Критический | Даёт злоумышленнику возможность быстро получить привилегированный доступ к системам и данным. |
| Секретные данные | Политики доступа, ролевые назначения, security-конфигурация | Искажение данных | Критический | Изменение правил доступа открывает путь к несанкционированным действиям и межсистемной компрометации. |
| Секретные данные | Полные резервные копии БД и выгрузки для восстановления | Потеря данных | Критический | Без резервных копий компания теряет возможность восстановить бизнес-критичные данные после инцидента. |

## Логика mindmap

- Центр: `Безопасность данных`
- Уровень 1: четыре категории данных
- Уровень 2: конкретные типы данных в каждой категории
- Уровень 3: риск и оценка
- Уровень 4: краткое объяснение оценки

## Замечания по выбору категорий

- `Публичные данные` не требуют строгой конфиденциальности, но чувствительны к искажению и устареванию.
- `Внутренние данные` не предназначены для внешнего раскрытия и критичны для управления, сопровождения и расследований.
- `Конфиденциальные данные` включают ПДн и сведения о клиентских и собственнических операциях.
- `Секретные данные` дают прямой доступ к системам или позволяют обойти меры защиты, поэтому для них выбран максимальный уровень важности.
