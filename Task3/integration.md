# Требования к интеграции Smart Home

## Цель

Интеграция с платформой Smart Home должна позволять собственнику открывать дверь и шлагбаум из мобильного приложения, а системе ЖКУ получать статусы устройств и события без прямого доступа партнёра во внутренние сервисы

## Основные требования

- внешний обмен с партнёром идет только через отдельный сервис `partner-smart-home-gateway`
- `partner-smart-home-gateway` размещается в `DMZ`
- на входе в контур ЖКУ ставится `WAF`
- для публичного контура включается общий `rate limiting`
- доступ к API партнёра идет через `firewall` с whitelist IP
- для интеграции используются `HTTPS`, `mTLS` и `OAuth2 Client Credentials`
- секреты, client secret и сертификаты хранятся во `Vault / KMS`
- события безопасности, ошибки интеграции и административные действия отправляются в `SIEM`

## Потоки

- мобильное приложение обращается во внутренний контур ЖКУ
- `tenant-core-app` получает контекст собственника и передает команды в `smart-home-access-service`
- `smart-home-access-service` работает с разрешениями и устройствами
- `partner-smart-home-gateway` обменивается командами и webhook только с платформой партнёра

## Нефункциональные требования

- webhook должен проходить валидацию подписи и источника
- повторная доставка событий должна обрабатываться идемпотентно
- ошибки партнёрского API не должны останавливать внутренний контур ЖКУ
- доступ к данным собственников и устройствам ограничивается tenant scoping
